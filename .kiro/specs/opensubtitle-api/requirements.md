## **要件定義書：`opensubtitles-api` を利用した字幕ダウンロード機能の改修**

### 1. 背景

現在、本アプリケーションはOpenSubtitlesから字幕をダウンロードする際に、Cloudflareのセキュリティにブロックされ、`469`エラーが頻発している。この問題を解決するため、より安定した公式APIである`opensubtitles-api`を利用するように、既存の字幕ダウンロード機能を改修する。

### 2. 目的

- Cloudflareによるブロックを回避し、字幕ダウンロードの成功率を向上させる。
- 公式APIを利用することで、将来的な仕様変更にも対応しやすくし、安定した運用を目指す。

### 3. スコープ

- **対象ファイル:** `opensubtitles.js`
- **改修内容:**
    - 現在利用している非公式な字幕取得処理を、`opensubtitles-api`ライブラリを利用した処理に置き換える。
    - APIキーなどの認証情報を、`.env`ファイルを通じて安全に管理できるようにする。
    - 字幕の検索、ダウンロード、およびファイル保存の一連の処理を、新しいAPIの仕様に合わせて実装する。

### 4. 機能要件

| No. | 機能 | 詳細 | 備考 |
| :-- | :--- | :--- | :--- |
| 1 | **APIクライアントの初期化** | `opensubtitles-api`ライブラリをインポートし、APIキーとUser-Agentを設定してクライアントを初期化する。 | User-Agentは、一般的なブラウザのものに偽装する。 |
| 2 | **字幕検索** | 映画・ドラマのIMDb ID、シーズン番号、エピソード番号を基に、指定された言語（英語）の字幕を検索する。 | 検索結果の中から、最も適切と思われる字幕を1つ選択する。 |
| 3 | **字幕ダウンロード** | 検索して取得した字幕情報（`file_id`など）を用いて、字幕ファイルのダウンロードURLを取得する。 | |
| 4 | **ファイル保存** | ダウンロードした字幕ファイルを、既存の命名規則に従ってローカルファイルシステムに保存する。 | |
| 5 | **エラーハンドリング** | APIからのエラーレスポンス（認証エラー、字幕が見つからない場合など）を適切に処理し、ログに出力する。 | |
| 6 | **設定管理** | OpenSubtitlesのAPIキーを`.env`ファイルで管理できるようにする。 | `.env.example`にも追記し、ユーザーが設定しやすいように案内する。 |

### 5. 非機能要件

| No. | 項目 | 詳細 |
| :-- | :--- | :--- |
| 1 | **セキュリティ** | APIキーは環境変数として扱い、コード内に直接記述しない。 |
| 2 | **パフォーマンス** | APIへのリクエストは、必要最小限に留める。 |
| 3 | **保守性** | コードはモジュール性を意識し、他の部分への影響を最小限に抑える。 |

### 6. 前提条件

- `opensubtitles-api`のnpmパッケージが利用可能であること。
- 有効なOpenSubtitlesのAPIキーが取得済みであること。
